Project Management System - University Project

This project involved the design and development of a project management system using C#, using a Waterfall methodology approach as each developement needed a previous part to be finished before it could be implemented, for exmaple team members role and functionality needed to be implemented before project managers as they needed to manage team members. The system was built to manage projects by utilizing a three-tiered user hierarchy consisting of members, managers, and administrators, each with different roles and permissions in-line with their titles.

A MySQL database was implemented to handle everything from user authentication, role management, to project data.

The user interface was dynamically adjusted based on the logged-in user’s role. Team members were limited to viewing and updating their assigned tasks, while managers had additional capabilities such as assigning tasks and monitoring progress. Administrators had full control over system-wide settings and user management.

Testing was conducted using multiple example user accounts to simulate real-world usage. This process revealed several usability issues, particularly related to interface clarity and functional bugs. These issues were addressed through iterative fixes and interface improvements.
